#### 果切订单主流程

1. 用户从购物车进入结算
2. 订单命中判定，系统判断订单是否命中果切
   * + 否：执行普通订单履约
     + 是：继续下一层判断
3. 系统盘点：果切订单是否走果切任务
   * + 是：执行普通订单履约
     + 否：系统自动按果切加时规则更新 ETA，用户侧展示“尽快送达"时间变更
4. 用户下单后，系统判断订单：配送单/自提单
   * + 配送单：
       1. 分拣任务冻结，等果切任务做完再激活
       2. 水产任务逻辑不变
       3. 果切任务仅在满足时间条件后，才在秤端 / pc端列表透出；未到时间则隐藏不展示。

参数定义：

用户预约时间：用户下单选择的预约送达时间的开始时间

保鲜时长：已有配置

库内分拣预留耗时：固定 6 分钟

果切前置缓冲时长：固定 30 分钟

计算公式：

任务可展示开始时间 = 用户预约时间 - 保鲜时长（多个品取最小的） - 库内分拣预留 6min - 前置缓冲 30min

透出判定规则：

系统当前时间 ≥ 任务可展示开始时间 时，果切任务在秤端正常可见、可加工；

当前时间未达该节点，任务不展示。

示例说明：

用户预约为 12:00-12:30， 取开始时间 12:00

按公式计算：12:00 往前扣减 30min（假设果切保鲜时长30min）+6min+30min = 10:54

10:54 起秤端可见该果切订单任务；10:54 前隐藏。

* + - 自提单：
      1. 先冻结果切/分拣/水产任务
      2. ①用户扫码提货 / ②库内兜底协助用户提货 时，仍旧先冻结分拣任务，只激活 果切/水产 任务，等果切任务完成再激活分拣任务。

1. 冻结状态任务，不支持人工改派
2. 果切任务一律正常下发
   * + 如有因订单取消、额外制造产生的库存，需要在秤上手动点「有库存，不用制造」
3. 分拣员履约，订单完成

#### 走果切任务，订单取消流程

1. 用户 / 客服发起取消申请
2. 系统判断：订单是否包含果切商品？
   * 不包含：执行普通订单取消逻辑。
   * 包含：进入果切订单取消判断逻辑。
3. 系统判断：果切任务是否到可处理时间？
   * 未到：允许客服和用户取消，取消成功后系统同步取消果切/分拣/水产任务。
   * 已到：继续判断任务执行状态。
4. 系统判断：继续判断果切任务状态？
   * 待处理：允许客服和用户取消，取消成功后系统同步取消全部关联任务。
   * 已完成：仅支持客服后台操作取消
     + 如果是整个订单未出库，系统会生成“转移任务-订单取消”任务给仓管
     + 如果是整个订单已出库，系统会生成“退货入库-订单取消”任务给仓管

#### 走果切任务，订单改约流程

1. 用户发起改约申请
2. 系统判断：订单是否包含果切商品？
   * 不包含：执行普通订单改约逻辑。
   * 包含：进入果切订单改约判断逻辑。
3. 系统判断：果切任务是否到可处理时间？
   * 未到：允许改约，改约成功后系统按用户新选择的履约时间重新分派果切/分拣/水产任务。
   * 已到：继续判断任务执行状态。
4. 系统判断：继续判断果切任务状态？
   * 待处理：允许客服和用户改约，改约成功后系统按用户新选择的履约时间重新分派果切/分拣/水产任务。
   * 已完成：只允许客服改约

#### 追单和补单

果切订单允许追单和补单，补单之后分拣任务、果切任务重新生成

补单场景：含果切任务订单，由果切任务控制，果切任务下发到秤上前允许补单，补单后果切任务重新生成。 ——果切任务处理时间可能会缩短，本版本暂不考虑。

补单挂起时要把果切任务也挂起

补单取消时要把果切任务也解挂起

补单完成时原来的果切任务和分拣任务一起删除

#### 果切任务列表、详情、和执行

用仓管员账号登陆，按站纬度下发果切任务

* 只要**任务被分配、任务状态发生变化**，后端必须主动推送客户端
* 客户端只做展示、执行、回传，不自己生成任务状态

**果切任务声音提醒：**

* 有新的果切任务时，声音提醒：有果切任务，请尽快处理！ （就是秤上能看到的时候声音提醒）
* 果切任务即将超时，声音提醒：果切任务已超时，请尽快处理！ 之后每隔2min提醒一次。 限时 5min支持配置，间隔2min支持配置
  + - （解释下意思：就是1个任务到秤上了之后， 5min还没完成，就语音提醒。 有任务超时后，间隔2分钟播一次语音。 10个任务都超时，也是2分钟播一次。）
* 两个声音提醒不管停留在那个页面都提醒

**任务展示样式：**

* 左侧卡片：展示订单号、任务号、预约时段、可处理时间、商品名称 、客户下单份数；按**果切任务可处理时间**从小到大展示，一样可处理时间排序随机。
* 右侧操作界面：商品主图、商品名称、客户下单份数、商品 ID、净重范围、标准净重
* 特殊展示逻辑：
  + - 果切商品：额外展示 使用原料和原料使用量，原料使用量展示规则如下：
      * 原料实际使用量不实时计算，直接取自 BOM 配置值，按原料单位区分展示逻辑：
      * 原料单位为份：
        + BOM 中配置为整数份，秤端原料使用量展示格式为「X 份」。
        + 例：成品 A × 成品数量 2 = 原料 B × 原料数量 1，则秤端展示原料 B 使用量为 1 份。
      * 原料单位为 kg：
        + BOM 中配置可为小数千克，称端自动换算为克展示，格式为「X g」。
        + 例：成品 A × 成品数量 1 = 原料 B×0.5kg + 原料 C×0.8kg，则秤端展示原料 B 使用量 500g、原料 C 使用量 800g。
    - 特殊加工品：额外展示， 提醒话术和处理方式。（因为果切不扫码了没办法判断果切前的品重量是否达标，所以先称重一下比较合适，防止人员拿错规格）
* 称重校验逻辑：
  + - 果切单品：按切好后净含量区间校验
    - 特殊加工品：按切前的净含量区间校验
* 支持：商品图片点击放大查看。

**交互：**

* 只支持自动称重，不支持合并称重，不支持免称重

**额外称重界面：**

一、触发条件

订单完成时，系统检测到「果切制造成品份数」＞「**实际称重份数**」（适用于西瓜切半等需多制造的场景），自动触发右侧抽屉，打开额外称重界面，要求果切员完成剩余份数的称重操作。

二、额外称重界面规则

1. 界面显示份数规则：当前需称重份数 = 本订单系统制造成品份数 - **实际称重份数**；

2. 列表排序规则：

按「任务+SKU」聚合展示，同一任务内同一SKU的多份需求需合并为一行；

按果切任务的完成时间升序排序。

三、称重及标签打印流程

1. 操作方式：需逐条完成称重，每完成1份称重，系统自动打印1个对应标签，同时当前行「需称重份数」减1；

2. 行项消失规则：

当当前行需称重份数减至0（全部称完），该行自动消失；

点击「重量报损」按钮并完成确认后，该行自动消失。

四、重量报损处理规则

1. 二次确认：触发「重量报损」操作时，系统需弹出二次确认弹窗，确认无误后方可执行报损；

2. 报损逻辑：报损完成后，对应果切商品库存自动扣减；同步执行库内流程——先将损坏损耗转移，再进行损溢出库操作；若涉及损溢出库，需上传实物图及销毁图（可上传默认图片替代）。

五、异常中断处理（数据持久化）

为避免异常中断导致数据丢失，系统需按以下规则同步数据，保障断点续办：

每完成1份称重并打印标签，前端立即将「任务号+SKUID+称重信息」同步至后端；

每完成1个SKU的重量报损，前端立即将「任务号+SKUID+报损份数」同步至后端；

异常中断后重新进入界面时，仅展示尚未完成称重/报损的未处理数据。

**异常场景：**

1. 订单取消：

弹窗提醒，提示语：订单XXXXXXX 订单已取消，商品ID：XXXX，商品名称：XXXXX，果切份数：XXX。 点「我知道了」关闭弹窗。

1. 单品bom未维护，但是在fmc中台维护了是果切品：

果切任务正常下发，点称重完成时提示异常不进行制造

在中台配置关闭果切任务推送，系统释放分拣任务，原来的果切任务状态更新成取消

#### 支持多原料造单成品制造和共享库存

**制造逻辑应用说明：**

**1. BOM 结构（两级转化）**

* 一级 BOM（份→kg，基础原料制造）
  + - 1 份 A = 3kg B， B的标重是 3000 g
    - 1 份 C = 2kg D， D的标重是 2000 g
* 二级 BOM（kg→混合果切）
  + - 1 份 E = 0.2\*B + 0.5\* D， E的标重是 300 g ，净重范围是 280g～350g

**2. 制造流程**

1. **先造半成品 B、D**
   * 触发：FMC 按安全库存定时下发制造任务
   * 制造 B：消耗 A，增加 B 库存
   * 制造 D：消耗 C，增加 D 库存
2. **再造混合果切 E**
   * 消耗：B、D 库存
   * 产出：E 成品
   * 系统自动扣减对应 kg 数量

**3. 示例（制造 1 份 E）**

1. 先造半成品
   * 造 6kg B = 消耗 2 份 A
   * 造 6kg D = 消耗 3 份 C
2. 再造 E
   * 1E = 0.2\* B + 0.5\* D
   * 实际消耗：0.2kg B + 0.5kg D
   * 库存变化：
     + B：6kg → 5.8 kg
     + D：6kg → 5.5 kg
     + E：+1 份
     + B和D按照bom中配置的比例扣，即使1E 称重了320g，B和D 就是按bom配置比例扣的，不会多扣

#### 果切临期报损

**果切临期定义**

一、果切商品制作完成并打印标签的时间为**成品生产时间**，**在库存还未扣减情况下**，以下情况系统判定为临期，在临期报损页面显示：

* + sku在业务配置中心配置了X小时临期，X小时之后显示出来这个品
  + 在兜底出清时间，显示出来没提醒的品
  + 只查当天的数据

**临期提醒规则**

1. 当系统存在临期果切商品时，立即触发全局提醒。
2. 声音提醒：播放语音提示 “有临期品，请及时临期报损！”**，提醒间隔为**5 分钟 / 次 ，间隔时长支持配置。
3. 页面提醒：临期报损旁，红色数字提醒，按行数统计，直至临期商品完成临期报损后自动消失。

**临期报损页面**

1. 页面顶部展示红色提示语：以下商品，超过品控可售标准不能出库，需要临期报损，以免因品质问题引起客诉！ 注意：剩余切出来非整个瓜/果需要集中在晚间出清时间手动做临期报损，不可过夜！
2. 列表展示字段：生产时间、商品 ID、商品名称、可用库存。

**报损操作规则**

1. 支持**单个报损**与**批量一键报损**两种操作方式。
2. 执行报损操作时，系统必须弹出二次确认框，确认后方可完成报损。
3. 报损完成后，对应果切商品库存扣减（同步库内**先临期转移，再损溢出库**），临期提醒同步更新。 如果库内库存不足，有多少报损多少。
